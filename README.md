# Lunar1ya — библиотека игр

Статический сайт без сборщика и внешних зависимостей. GitHub Pages публикует файлы репозитория напрямую.

## Структура

```text
.
├── index.html                  # только разметка страницы
├── css/styles.css              # все стили и адаптивная вёрстка
├── js/games.js                 # список игр и пути к обложкам
├── js/app.js                   # фильтры, сортировка, карточки и статистика
├── assets/
│   ├── character/              # персонаж Lunar1ya с ушками хорька в WebP
│   └── covers/                 # обложки игр: JPG (сюжетные) и SVG (тематические)
└── tools/
    ├── make_vector_covers.py   # генератор тематических SVG-обложек
    └── add_cover_titles.py     # добавляет названия на JPG-обложки
```

## Как изменить список игр

1. Откройте `js/games.js`.
2. Добавьте или измените объект в массиве `GAMES`.
3. Для своей обложки добавьте путь в `COVER_ART`, например:

```js
"Название игры": "assets/covers/nazvanie-igry.jpg",
```

Поддерживаемые статусы: `new`, `playing`, `done`, `planned`. Дополнительные теги: `mp`, `coop`.

## Локальный просмотр

Из корня репозитория:

```bash
python3 -m http.server 8000
```

Затем откройте <http://localhost:8000>. Сервер нужен, потому что ES-модули (`js/app.js` и `js/games.js`) браузеры блокируют при открытии `index.html` напрямую через `file://`.

## Перегенерация SVG-обложек

После изменения цветовой темы или персонажа:

```bash
python3 tools/make_vector_covers.py
python3 tools/add_cover_titles.py
```

Первый скрипт перегенерирует SVG-обложки, второй добавляет названия на JPG-обложки. Оба используют локальные файлы и не требуют сторонних Python-библиотек.
